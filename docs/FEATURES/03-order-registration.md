# 03. 시료 주문 (접수)

출처: PDF p.14~15 (시료 주문)

## 개요

고객이 시료를 요청하면 주문 담당자가 주문을 생성(예약)한다. 이 메뉴는 새 주문을 시스템에 접수하는
기능만 다루며, 승인/거절은 [주문 승인/거절](04-order-approval-rejection.md)에서 처리한다.

## 시료 예약 (주문 생성)

- 고객이 원하는 시료와 수량을 주문으로 등록한다.
- 등록 시점에 주문 상태는 `RESERVED`이다.

### 입력값

| 필드 | 설명 |
|---|---|
| 시료 ID | 등록된 시료 중에서 선택 (미등록 시료는 주문 불가) |
| 고객명 | 자유 텍스트 |
| 주문 수량 | 1 이상의 정수 |

### 처리 결과

- 주문번호(예: `ORD-YYYYMMDD-NNNN` 형식)를 자동 채번한다.
- 생성된 주문의 상태는 `RESERVED`로 초기화한다.
- 이 단계에서는 재고 확인/차감을 하지 않는다 (재고 확인은 승인 단계에서 수행, [04](04-order-approval-rejection.md) 참고).

## 검증 규칙

- 존재하지 않는 시료 ID는 거부하고 재입력을 요구한다.
- 주문 수량은 1 이상의 정수만 허용한다.
- 고객명은 공백일 수 없다.

## 데이터 영속성 연계

- 생성된 주문은 애플리케이션 재시작 후에도 유지되어야 한다.

## 참고 예시 UI (PDF p.15, 자유롭게 재구성 가능)

```
[2] 시료 주문
시료 ID   > S-003
고객명    > 삼성전자 파운드리
주문 수량 > 200

입력 내용 확인
시료   SiC 파워기판-6인치 (S-003)
고객   삼성전자 파운드리
수량   200 ea

[Y] 예약 접수  [N] 취소
선택 > Y

예약 접수 완료.
주문번호  ORD-20260416-0043
현재 상태 RESERVED

※ 재고 확인은 [3] 승인 메뉴에서 직접 진행하세요.
```
