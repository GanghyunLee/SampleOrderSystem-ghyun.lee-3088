# 02. 시료 관리

출처: PDF p.12~13 (시료 관리)

## 개요

시료(Sample)는 이 시스템의 가장 기본이 되는 단위이다. 각 시료는 고유한 이름과 속성을 가지며,
**시스템에 등록된 시료만 주문 가능**하다.

## 도메인 속성

| 필드 | 설명 |
|---|---|
| 시료 ID | 고유 식별자 (예: `S-001`) |
| 이름 | 시료명 (예: "실리콘 웨이퍼-8인치") |
| 평균 생산시간 | 시료 1개를 생산하는 데 걸리는 평균 시간 (분 단위) |
| 수율 | `정상적인 시료 수 / 총 생산 시료 수` (0~1 범위 소수). 예: 100개 생산 중 90개 정상 = 0.9 |
| 현재 재고 | 등록 시점 이후 누적된 현재 재고 수량 (조회 시 함께 표시, 별도 입력 항목 아님) |

## 하위 기능

### 시료 등록

- 새로운 시료를 시스템에 추가한다.
- 입력값: 시료 ID, 이름, 평균 생산시간, 수율.
- 시료 ID는 중복될 수 없다 (중복 시 등록 거부 또는 재입력 요구).
- 수율은 `0 < 수율 ≤ 1` 범위를 벗어나면 입력을 거부한다.
- 등록 직후 초기 재고는 0으로 시작한다.

### 시료 조회

- 등록된 모든 시료 목록을 확인한다.
- 각 시료의 현재 재고 수량을 함께 표시한다.
- 목록이 많을 경우 페이지네이션 등 UX는 자유롭게 결정한다 (PDF 예시는 "외 N종 [N] 다음페이지" 형태).

### 시료 검색

- 이름 등 속성으로 특정 시료를 검색한다.
- 최소 요구: 이름 부분 일치 검색.
- (구현 시 결정 필요) 시료 ID 검색, 수율/재고 범위 검색 등 확장 여부는 팀 재량.

## 데이터 영속성 연계

- 시료 등록/재고 변동은 애플리케이션 재시작 후에도 유지되어야 한다 (`DataPersistence` PoC의 JSON 저장 구조 참고).

## 예외 처리

- 존재하지 않는 시료 ID로 주문하려는 시도는 이 화면이 아니라 [주문 접수](03-order-registration.md) 단계에서 거부한다.
- 시료 등록 시 필수값 누락/형식 오류는 재입력을 요구한다.
