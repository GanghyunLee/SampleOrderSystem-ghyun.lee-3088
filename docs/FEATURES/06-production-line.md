# 06. 생산 라인

출처: PDF p.7, p.20~21 (시스템 개요 / 생산라인)

## 개요

생산라인에 대한 정보를 표시한다. 생산라인은 시료 하나를 생산하는 설비 흐름이며, **하나의 생산 라인은
한 번에 시료 하나씩** 생산한다 (단일 라인, 순차 처리). 주문이 들어온 시료에 대해서만 생산한다.

## 생산 트리거

- [주문 승인](04-order-approval-rejection.md)에서 재고 부족으로 승인된 주문은 자동으로 생산 큐에 등록되고
  주문 상태는 `PRODUCING`이 된다.

## 생산량 계산 규칙

| 항목 | 계산식 |
|---|---|
| 부족분 | `주문 수량 - 현재 재고` |
| 실 생산량 | `ceil(부족분 / 수율)` — 수율(불량률)을 고려하여 부족분보다 더 많이 생산 |
| 총 생산 시간 | `평균 생산시간 × 실 생산량` |

- 수율과 평균 생산시간은 해당 주문의 시료(Sample) 속성값을 사용한다 ([시료 관리](02-sample-management.md) 참고).
- 생산 완료 시 주문 상태를 `PRODUCING` → `CONFIRMED`로 변경한다.
- 생산으로 만들어진 수량 중 부족분을 초과하는 여분(수율 보정으로 인한 초과 생산량)은 해당 시료의 재고로 편입한다.

## 생산 현황 표기

- 현재 생산 중인 시료에 대한 정보를 표기한다.
- 표기할 정보 수준은 자율적으로 결정한다. 예: 주문 정보, 현재까지의 생산량, 진행률(%), 완료 예정 시각.

## 대기 주문 확인 (생산 큐)

- 생산 라인의 대기열(생산 큐)을 이용해 생산 작업을 대기 중인 목록을 출력한다.
- **스케줄링 전략: FIFO(선입선출)** — 승인(등록)된 순서대로 생산한다.
- 대기 항목별로 주문량/부족분/실생산량/예상 완료 시각 등을 표시할 수 있다.

## 예외 처리

- 생산 큐가 비어 있을 때는 "대기 중인 생산 작업 없음"을 표시한다.
- 생산 라인이 다중으로 확장되는 시나리오는 본 과제 범위 밖이다 (단일 라인 전제, `docs/PRD.md` §9 참고).
