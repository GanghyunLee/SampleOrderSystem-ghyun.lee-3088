# 04. 주문 승인/거절

출처: PDF p.16~17 (주문 승인/거절)

## 개요

접수된 주문(`RESERVED`) 목록을 확인하고, 생산 라인 담당자가 특정 주문에 대해 승인 또는 거절을 결정한다.

## 접수된 주문 목록

- `RESERVED` 상태의 주문 목록을 표시한다.
- 표시 항목 예: 주문번호, 고객명, 시료명, 수량, 상태.

## 주문 승인

접수된 특정 주문에 대해 승인을 처리하면, **재고 상황에 따라 자동으로 2가지 방식 중 하나로 분기**한다.

| 조건 | 처리 |
|---|---|
| 재고가 주문 수량 이상 (충분) | 주문을 즉시 `CONFIRMED` 상태로 전환하고, 해당 수량만큼 재고를 차감한다 |
| 재고가 주문 수량 미만 (부족) | 생산 라인에 자동으로 등록(생산 큐에 추가)하고, 주문 상태를 `PRODUCING`으로 전환한다 |

- 부족분 계산: `부족분 = 주문 수량 - 현재 재고`
- 재고가 부족한 경우 기존 재고를 먼저 소진하는지 여부는 구현 시 결정 필요 (권장: 기존 재고는 전량 차감하여 0으로 만들고,
  부족분만큼을 생산 큐에 등록 — [생산 라인](06-production-line.md)의 "실 생산량" 계산과 일치시킨다).
- 승인 시 재고 확인 결과(부족분, 실생산량 등 참고 정보)를 화면에 함께 보여줄 수 있다 (PDF 예시 참고).

## 주문 거절

- 접수된 특정 주문에 대해 거절을 처리하면 즉시 `REJECTED` 상태로 전환한다.
- `REJECTED`는 정상 흐름 외의 상태이며, 이후 모니터링/생산/출고 어디에서도 다루지 않는다.

## 상태 전이 요약

```
RESERVED --(승인, 재고 충분)--> CONFIRMED
RESERVED --(승인, 재고 부족)--> PRODUCING  (생산 큐에 등록)
RESERVED --(거절)-------------> REJECTED
```

## 예외 처리

- `RESERVED` 상태가 아닌 주문에 대해 승인/거절을 시도하면 거부한다 (이미 처리된 주문).
- 존재하지 않는 주문번호 선택 시 오류 메시지를 출력한다.
