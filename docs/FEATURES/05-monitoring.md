# 05. 모니터링

출처: PDF p.18~19 (모니터링)

## 개요

주문에 따른 시료 재고에 대한 정보를 모니터링하는 기능. 담당자가 현재 시스템 상태를 한눈에 파악할 수
있도록 구성한다.

## 하위 기능

### 주문량 확인

- 현재 상태별(`RESERVED` / `CONFIRMED` / `PRODUCING` / `RELEASE`) 주문 건수를 확인한다.
- `REJECTED`는 유효한 주문이 아니므로 **집계에서 제외**한다.

### 재고량 확인

- 각 시료별 현재 재고 수량을 확인한다.
- 주문 대비 재고 수량에 따라 상태를 함께 표기한다.

| 상태 | 의미 |
|---|---|
| 여유 | 주문 대비 재고 충분 상태 |
| 부족 | 주문 대비 재고 수량 부족 상태 |
| 고갈 | 재고 수량이 0인 상태 |

- "주문 대비"의 기준(어떤 미출고 주문 합계와 비교하는지)은 구현 시 명확히 정의해야 한다.
  권장 정의: 해당 시료에 대해 아직 출고되지 않은 유효 주문(`RESERVED`+`CONFIRMED`+`PRODUCING`)의 수량 합계 대비
  현재 재고 비율로 여유/부족/고갈을 판단한다.
- 잔여율(%) 등 보조 지표는 자유롭게 추가 가능하다 (PDF 예시 화면 참고).

## 예외 처리

- 등록된 시료/주문이 없을 때는 "표시할 데이터 없음"과 같은 안내를 출력한다 (에러가 아님).

## 데이터 모니터링 Tool과의 관계

- 이 메뉴는 최종 사용자(담당자)를 위한 도메인 모니터링(주문/재고 요약)이다.
- 미션1의 `DataMonitor` PoC는 저장소(JSON 파일) 자체의 상태를 실시간 조회하는 관리자 도구로,
  본 기능 구현 시 참고할 수 있는 데이터 접근 패턴을 제공한다.
